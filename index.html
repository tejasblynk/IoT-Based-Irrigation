üìñ Project Abstract

This project implements an intelligent plant watering system using the ESP8266 NodeMCU. It features a dual-mode operation: an Automatic Mode that triggers watering based on real-time capacitive soil moisture data, and a Manual Mode allowing remote control via the Sinric Pro IoT platform. The system includes a flicker-free LCD dashboard for live status monitoring and integrates with voice assistants like Alexa and Google Home.

üîå Circuit Diagram

The circuit consists of the ESP8266 as the central controller, interfacing with the moisture sensor via analog input, controlling the relay module through a digital GPIO pin, and displaying real-time data on an I2C LCD screen.

‚ú® Key Implementations

  • Flicker-Free Display: Custom logic updates LCD only when values change, preventing screen refreshing artifacts and improving readability.
  • Hysteresis Logic: Auto-watering triggers at 680 ADC (Dry threshold) and stops below 680 to prevent rapid relay switching and extend hardware lifespan.
  • IoT Integration: Uses Sinric Pro API for real-time bi-directional communication with 5-second update intervals for responsive remote control.
  • Safety Override: Manual control via app automatically disables Auto Mode to prevent logic conflicts and ensure user control priority.
  • Voice Assistant Support: Full integration with Amazon Alexa and Google Home for hands-free operation.
  • Real-time Monitoring: Live moisture percentage display with automatic status updates on IoT dashboard.

üíª Firmware Logic (C++/Arduino)

The system runs on custom firmware written in C++ using the Arduino framework. Below is the core logic implementation showcasing the auto-watering algorithm and LCD update mechanism. 

/* ==========================================
   SMART IRRIGATION SYSTEM - CORE LOGIC
   ==========================================
   Authors: Tejas Niranjan & Kayansh Gupta
   Platform: ESP8266 NodeMCU
   Framework: Arduino / PlatformIO
   ========================================== */

#include <ESP8266WiFi.h>
#include <SinricPro.h>
#include <LiquidCrystal_I2C.h>

// Pin Definitions
#define RELAY_PIN 14        // D5 (GPIO14)
#define MOISTURE_PIN A0     // Analog Input
#define LCD_ADDR 0x27       // I2C Address

// Threshold Constants
#define NEITHER_DRY_OR_WET 680
#define UPDATE_INTERVAL 5000

// Auto Watering Control Function
void autoWateringControl(int rawValue) {
  if (!autoModeEnabled) return;

  // DRY Condition ‚Üí Activate Pump
  if (rawValue > NEITHER_DRY_OR_WET && !autoPumpRunning) {
    digitalWrite(RELAY_PIN, HIGH);
    autoPumpRunning = true;
    pumpSwitch.sendPowerStateEvent(true);
    Serial.println("[AUTO] Pump ON - Soil is DRY");
  }
  // WET Condition ‚Üí Deactivate Pump
  else if (rawValue <= NEITHER_DRY_OR_WET && autoPumpRunning) {
    digitalWrite(RELAY_PIN, LOW);
    autoPumpRunning = false;
    pumpSwitch.sendPowerStateEvent(false);
    Serial.println("[AUTO] Pump OFF - Soil is WET");
  }
}

// LCD Update with Anti-Flicker Logic
void updateLCD(int moisturePercent, bool forceUpdate) {
  // Only update if value changed or forced
  if (moisturePercent != lastDisplayedMoisture || forceUpdate) {
    // Line 1: Moisture Reading
    lcd.setCursor(0, 0);
    lcd.print("Moist: ");
    lcd.print(moisturePercent);
    lcd.print("%   "); // Padding for overwrite
    
    // Line 2: System Status
    lcd.setCursor(0, 1);
    if (autoModeEnabled) {
      lcd.print("Mode: AUTO     ");
    } else {
      lcd.print("Mode: MANUAL   ");
    }
    
    lastDisplayedMoisture = moisturePercent;
  }
}

// Main Loop
void loop() {
  SinricPro.handle();
  
  // Read Moisture Sensor
  int rawValue = analogRead(MOISTURE_PIN);
  int moisturePercent = map(rawValue, 1023, 0, 0, 100);
  
  // Execute Auto Logic
  autoWateringControl(rawValue);
  
  // Update Display
  updateLCD(moisturePercent, false);
  
  delay(100);
}

üìä System Workflow

1. Sensor Reading: Capacitive moisture sensor continuously measures soil water content.
2. Data Processing: Raw ADC values converted to percentage (0-100%) via calibrated mapping.
3. Mode Selection: User chooses Auto or Manual mode via Sinric Pro app/voice command.
4. Decision Logic: Auto mode triggers pump based on hysteresis threshold (680 ADC).
5. Actuation: Relay switches pump ON/OFF with status feedback to IoT platform.
6. Display Update: LCD shows real-time moisture % and active mode.
